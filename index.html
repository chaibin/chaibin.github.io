MySQL 百万级数据量分页查询方法及其优化

2019-07-19

原生查询

试验 200W数据表

select * from user_test  limit 1000000,20;
执行时间 1.7s
select * from user_test  limit 1500000,20;
执行时间 2.6s

随偏移量越大，mysql查询效率越低，执行全表扫描

优化方案

覆盖索引
select id from user_test  limit 1000000,20;
执行时间： 0.2
select id from user_test  limit 1500000,20;
执行时间： 0.38
select id from user_test order by  id  asc limit 1500000,20;
执行时间 0.6 添加排序后更慢
select * from user_test where id >= (select id from user_test  limit 1500000,1)  limit 10;
执行时间 0.7 还是有点慢
